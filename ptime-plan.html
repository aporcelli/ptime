Lógica de Negocio
y Precios
Algoritmo de cálculo escalonado, manejo de decimales e inyección de configuración

Reglas de Precios

💰
Tramo 1
$35 / hora
Aplica desde la hora 0 hasta las primeras 20 horas acumuladas en el proyecto.
📈
Tramo 2
$45 / hora
Aplica a partir de la hora 20.01 en adelante. La transición es precisa al decimal.
⚙️
Configurable
Por proyecto
Cada proyecto puede sobreescribir los precios globales desde el módulo Admin.

Algoritmo de Cálculo (TypeScript)

// lib/pricing/calculateHoursAmount.ts
// ─────────────────────────────────────────────────────────────
// Principio: función pura, sin efectos secundarios, totalmente testeable

interface PricingConfig {
  precioBase: number;       // default: 35
  precioAlto: number;       // default: 45
  umbralHoras: number;      // default: 20
}

interface CalculationResult {
  montoTotal: number;
  precioAplicado: number;   // precio predominante del tramo
  desglose: {
    horasEnTramo1: number;
    horasEnTramo2: number;
    montoTramo1: number;
    montoTramo2: number;
  };
}

export function calculateHoursAmount(
  horasNuevas: number,            // Horas del nuevo registro (puede ser decimal)
  horasAcumuladasAntes: number,   // Total de horas del proyecto ANTES de este registro
  config: PricingConfig
): CalculationResult {
  const { precioBase, precioAlto, umbralHoras } = config;

  // Redondear a máximo 4 decimales para evitar errores de punto flotante
  const horas = Math.round(horasNuevas * 10000) / 10000;
  const acum = Math.round(horasAcumuladasAntes * 10000) / 10000;

  const horasEnTramo2Antes = Math.max(0, acum - umbralHoras);
  const horasEnTramo1Antes = acum - horasEnTramo2Antes;

  const capacidadRestanteTramo1 = Math.max(0, umbralHoras - horasEnTramo1Antes);

  const horasEnTramo1 = Math.min(horas, capacidadRestanteTramo1);
  const horasEnTramo2 = Math.max(0, horas - horasEnTramo1);

  const montoTramo1 = Math.round(horasEnTramo1 * precioBase * 100) / 100;
  const montoTramo2 = Math.round(horasEnTramo2 * precioAlto * 100) / 100;
  const montoTotal  = Math.round((montoTramo1 + montoTramo2) * 100) / 100;

  // El precio "aplicado" es el del tramo donde cayó la mayoría de las horas
  const precioAplicado = horasEnTramo2 > horasEnTramo1 ? precioAlto : precioBase;

  return {
    montoTotal,
    precioAplicado,
    desglose: { horasEnTramo1, horasEnTramo2, montoTramo1, montoTramo2 }
  };
}

// ── EJEMPLO DE USO ──────────────────────────────────────────
// Proyecto con 18h acumuladas, nuevo registro de 4h
// calculateHoursAmount(4, 18, { precioBase: 35, precioAlto: 45, umbralHoras: 20 })
// → { montoTotal: 160, desglose: { horasEnTramo1: 2 ($70), horasEnTramo2: 2 ($90) } }
✅
Precisión Decimal El algoritmo usa multiplicación por 10000 + redondeo antes de cada suma para eliminar errores de punto flotante. Los montos finales siempre tienen 2 decimales exactos. Se recomienda agregar tests unitarios con Vitest para todos los edge cases (ej. 19.75h + 1.5h).

Inyección de Configuración (Server Action)

// app/actions/pricing.ts
'use server'
import { getSheetData } from '@/lib/sheets/client'
import { configSchema }  from '@/lib/schemas/config'

let _configCache: PricingConfig | null = null;
let _cacheTs: number = 0;
const CACHE_TTL_MS = 5 * 60 * 1000; // 5 minutos

export async function getPricingConfig(): Promise<PricingConfig> {
  const now = Date.now();
  if (_configCache && (now - _cacheTs) < CACHE_TTL_MS) return _configCache;

  const rows = await getSheetData('Configuraciones!A:B');
  const raw  = Object.fromEntries(rows.map(([k, v]) => [k, v]));

  _configCache = configSchema.parse({
    precioBase:   Number(raw.precio_base_global  ?? 35),
    precioAlto:   Number(raw.precio_alto_global  ?? 45),
    umbralHoras:  Number(raw.umbral_horas_global ?? 20),
  });
  _cacheTs = now;
  return _configCache;
}

1
Request entrante → middleware.ts intercepta toda ruta bajo /(dashboard) y /api protegidas.
2

3

Módulo Usuario — Formulario de Carga de Horas

🖊️
El formulario utiliza react-hook-form con resolución Zod para validación en tiempo real. Cada campo se valida on-blur y on-change. Un componente PricingPreview calcula el monto estimado al vuelo sin llamadas al servidor.
// Esquema Zod del formulario de horas
export const hourFormSchema = z.object({
  proyecto_id: z.string().uuid({ message: "Selecciona un proyecto válido" }),
  tarea_id:    z.string().uuid({ message: "Selecciona una tarea" }),
  fecha:       z.string().regex(/^\d{4}-\d{2}-\d{2}$/, "Formato YYYY-MM-DD"),
  horas:       z.number()
                 .min(0.25, "Mínimo 15 minutos (0.25)")
                 .max(24, "No puedes cargar más de 24h por registro")
                 .multipleOf(0.25, "Incrementos de 15 minutos"),
  descripcion: z.string()
                 .min(10, "Descripción mínima de 10 caracteres")
                 .max(500, "Máximo 500 caracteres")
                 .transform(val => DOMPurify.sanitize(val.trim())),
});

Características UX del Formulario

  • Selector de horas con pasos de 0.25 (15 min), con opción de texto libre para decimales exactos.
  • Preview de precio en vivo: muestra "~$52.50" mientras se escribe, usando el hook usePricingPreview.
  • Estado optimista: el registro aparece en la lista inmediatamente, con indicador de sincronización.
  • Detección de solapamiento: advertencia si hay registros en la misma fecha para el mismo proyecto.

Seguridad · Guía
de 5 Puntos Clave
Implementación rigurosa de seguridad a pesar de usar Google Sheets como backend
01
Credenciales nunca en el cliente
Las API Keys de Google Service Account y NEXTAUTH_SECRET viven exclusivamente en variables de entorno sin prefijo NEXT_PUBLIC_. El cliente de Google Sheets es un singleton server-side. Ninguna credencial cruza al bundle de cliente.
// lib/sheets/client.ts — Solo ejecuta en servidor
import { google } from 'googleapis';
const auth = new google.auth.GoogleAuth({
  credentials: {
    client_email: process.env.GOOGLE_SERVICE_ACCOUNT_EMAIL,
    private_key: process.env.GOOGLE_PRIVATE_KEY?.replace(/\\n/g,'\n'),
  },
  scopes: ['https://www.googleapis.com/auth/spreadsheets'],
});
02
Validación en dos capas (Zod)
Toda entrada de datos es validada dos veces: en el cliente para UX y en el servidor (Server Action) como fuente de verdad. No confiar en validaciones de cliente. Zod rechaza tipos inesperados, strings maliciosos y rangos fuera de límite.
// app/actions/hours.ts
'use server'
export async function createHour(rawData: unknown) {
  const session = await auth(); // Verificar auth PRIMERO
  if (!session) throw new Error('UNAUTHORIZED');

  const parsed = hourFormSchema.safeParse(rawData);
  if (!parsed.success) {
    return { error: parsed.error.flatten() };
  }
  // Solo aquí se accede a Sheets
}
03
Sanitización Anti-XSS
Todos los campos de texto libre (descripción, nombres) pasan por DOMPurify antes de ser almacenados y al ser renderizados. Se usa la función sanitize() centralizada. Nunca usar dangerouslySetInnerHTML con datos de usuario.
// lib/utils/sanitize.ts
import createDOMPurify from 'isomorphic-dompurify';
const DOMPurify = createDOMPurify();

export function sanitize(dirty: string): string {
  return DOMPurify.sanitize(dirty, {
    ALLOWED_TAGS: [], // Sin HTML permitido
    ALLOWED_ATTR: [],
  }).trim();
}
04
Sesiones HttpOnly + CSRF
NextAuth configura cookies HttpOnly, Secure y SameSite=Lax automáticamente. Los Server Actions de Next.js incluyen protección CSRF nativa mediante el mecanismo de Same-Origin verification. Configurar NEXTAUTH_SECRET con al menos 32 bytes aleatorios.
// auth.ts (NextAuth v5)
export const { handlers, auth, signIn, signOut } = NextAuth({
  secret: process.env.NEXTAUTH_SECRET,
  session: { strategy: "jwt", maxAge: 8 * 60 * 60 }, // 8h
  cookies: {
    sessionToken: {
      options: { httpOnly: true, sameSite: 'lax', secure: true }
    }
  }
});
05
Autorización por Roles (RBAC) en Middleware
El archivo middleware.ts protege rutas según el rol del token JWT. Las rutas /admin/* requieren rol ADMIN. Las rutas /horas/* requieren al menos USER. Cualquier intento de acceso sin el rol adecuado retorna redirect o 403 antes de que el componente se ejecute. Los Server Actions repiten esta verificación internamente.
// middleware.ts
import { auth } from '@/auth';
export default auth((req) => {
  const isAdminRoute = req.nextUrl.pathname.startsWith('/admin');
  const role = req.auth?.user?.role;
  if (!req.auth) return Response.redirect(new URL('/login', req.url));
  if (isAdminRoute && role !== 'ADMIN') return new Response(null, { status: 403 });
});
export const config = { matcher: ['/(dashboard)/:path*', '/admin/:path*'] };

Comandos de Inicio Rápido

# 1. Crear proyecto
npx create-next-app@latest ptime --typescript --tailwind --app --src-dir=false

# 2. Instalar dependencias críticas
npm install next-auth@beta googleapis zod uuid framer-motion
npm install recharts @react-pdf/renderer date-fns @tanstack/react-query
npm install isomorphic-dompurify @types/dompurify
npm install react-hook-form @hookform/resolvers

# 3. Instalar Shadcn/UI
npx shadcn@latest init
npx shadcn@latest add button input form select table badge card dialog

# 4. Configurar Service Account Google
#    → IAM → Service Accounts → Crear → Descargar JSON
#    → Compartir Spreadsheet con el email de la Service Account

# 5. Variables de entorno
cp .env.example .env.local
# Completar GOOGLE_SERVICE_ACCOUNT_EMAIL, GOOGLE_PRIVATE_KEY, GOOGLE_SPREADSHEET_ID

# 6. Dev server
npm run dev
